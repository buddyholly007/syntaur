# Gemini reviews — 2026-04-30 (v0.6.5 ship)

This file captures every Gemini code-review finding from the v0.6.5
ship session along with its triage verdict, file:line evidence, and
resolution. The `review_triage` stage in syntaur-ship reads this format
on every deploy and refuses to ship if any finding lacks the three
required fields (verdict + evidence + resolution). The intent is to
make "Gemini flagged X but it's preexisting/unrelated" structurally
impossible without a logged decision.

Format: each finding is a fenced ```finding ...``` block with the
required keys: `id`, `reviewer`, `file`, `claim`, `verdict`, `evidence`,
`resolution`. Verdict is `TRUE | FALSE | DEPENDENT`. Evidence must
reference a concrete `file:line`. Resolution starts with `fix-in-place`,
`tracked: <path>`, or `wont-fix: <reason>`.
